项目管理-沟通管理

项目管理 十大知识领域 沟通管理
项目管理
发布日期:   2021-05-12

沟通管理
Project Communication Management

项目经理绝大多数实践(90%) 都用玉团队成员和其他干系人的沟通

管理过程3个

规划沟通管理
Plan Communication

根据干系人的信息和要求以及组织的可用资产情况,制定合理的项目沟通方式和计划的过程

输入

项目管理计划
干系人登记册
事业环境因素 组织过程资产

工具技术

沟通需求分析
会议
沟通技术
谈话, 会议,书面文件,在线查询资料
选择因素:
信息需求的紧迫性
技术的可用性
易用性: 培训计划
项目环境
信息的敏感性和保密性
沟通障碍
缺乏清晰的沟通渠道
发送者与接受者存在屋里距离
沟通双方彼此技术语言不通
分散注意力的环境因素(噪声)
有害的态度(敌对, 不信任)
权力游戏, 直流信息,隐藏议程和敌对情绪
沟通模型
沟通方法
交互式沟通
两方或多方多向信息交互 (会议,电话,即时通信)
推式沟通
信息发送给特定接收方 (信件,邮件.)
拉式沟通
信息量很大,或受众很多(企业公告, 电子课堂)

输出

沟通管理计划
项目干系人沟通要求 (什么人)
需要沟通的信息, 包括语言个事内容和详细程度 (以什么形式)
发布信息的原因
发布信息以及告知(包含如果需要做出回应)的时限和频率
负责沟通项管信息的人员
负责授权保密信息发布的人员
将接收信息的个人或小组
传递信息的技术或方法, 入备忘录,电子邮件,新闻稿等
为沟通活动分配的资源,包含时间与预算
问题升级程序, 用于规定下层员工无法解决问题时的上报时限和上报路径
随项目进展,对沟通管理计划进行更新与优化的方法
通用术语表
项目信息流向图, 工作程程,报告清单
沟通制约因素, 通常来自特定的法律法规, 技术要求和组织政策等
项目文件更新

管理沟通
Manage Communication

根据沟通管理计划,生成,收集,分发,存储,检索以及最终处置项目信息的过程

输入

沟通管理计划
工作绩效报告(监控项目工作的输出)
事业环境因素, 组织过程资产

工具技术

沟通技术,沟通模型,沟通方法
信息管理系统
报告绩效
报告绩效是指收集和发布绩效信息,包括状态报告, 进展测量以及预测结果
详尽的报告
对过去绩效的分析
本报告期完成的工作
风险和问题的当前状态
本报改期被批准的变更汇总
下一个报告期要完成的工作
项目预测分析, 包括实践成本
需要审查和讨论的其他相关信息

输出

项目沟通
项目管理计划更新, 项目文件更新, 组织过程资产更新

控制沟通
Control COmmunication

整个项目生命周期中对沟通进行监督和控制的过程,以确保满足项目干系人对信息的需求

输入

项目管理计划
项目沟通,问题日志
工作绩效数据,组织过程资产

工具技术

信息管理系统
专家判断
会议

输出

工作绩效信息
变更请求
项目管理计划更新,项目文件更新,组织过程资产更新

非正式的沟通有助于关系的融洽

发展更好的沟通技能
