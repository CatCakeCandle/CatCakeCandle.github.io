    人力资源管理

    管理过程(4个)

    规划人力资源管理
    Personnel Human Resource Plan

    定义:识别和记录项目角色,之色,所需技能,报告关系,并编制人员配置管理计划的过程<br/>作用: 建立项目角色于职责, 项目组织图, 以及包含人员招募和前三时间表的人员配备管理&nbsp;<br/>

    输入

    项目管理计划
活动资源需求
事业环境因素 组织过程资产

    工具技术

    组织图和职位描述, 组织理论
    责任分配矩阵(RAM)
    RACI 矩阵


    层次结构图(OBS)
    文本格式
    人际交际
    会议,专家判断

    输出

    人力资源管理计划
    角色于职责 (所需岗位,技能, 角色,职权,职责, 能力)
项目组织图 (以图形方式展示项目团队成员及其报告关系)
人员配备管理计划 (所需人员,每个团队成员的时间段)

    组件项目团队
    Acquire Project Team

    确认人力资源的可用情况,并为开展项目活动而组件团队的过程<br/>作用: 指导团队选择和职责分配,组件一个成功的团队

    输入

    人力资源管理计划
事业环境因素 组织过程资产

    工具技术

    预分派
    谈判
    与职能经理
    招募
    招聘
    虚拟团队
    优点
    有可能在不同地理位置的员工之间组件团队
为项目团队增加特殊技能, 即使相应的专家不在统一地理区域.
将在家办公的员工纳入团队
在工作班次, 工作小时或工作日不同的员工之间组件团队
将行为不变者或者残疾人纳入团队
执行那些原因会因差旅费用过高而被否决的项目
    缺点
    可能产生误解, 有柜里干,团队成员之间难以分享知识和经验, 采用通信技术会有成本
需要重点管理, 注重沟通,&nbsp;
多标准决策技术

    输出

    项目人员分派
资源日历
项目管理计划更新

    建设项目团队
    Develop Project Team

    提高工作能力,促进团队成员互动, 改善团队整氛围,以提高项目绩效的过程<br/>作用: 改进团队写作,增强人际技能, 激励团队成员,降低人员离职率, 提升整体项目绩效.

    目标

    提高项目团队成员的个人技能
提升信任感,凝聚力, 以提高士气,降低冲突,促进团队合作
共享经验和知识

    阶段

    形成阶段 Forming
    刚组建,不熟悉, 在相互了解,角色定位
    震荡阶段 Storming
    冲突,不和谐, &nbsp;项目经理最值得花时间管控的阶段
    规范阶段 Norming
    相互信任,形成凝聚力,有合作意识&nbsp;
    发挥阶段 Performing
    相互理解,高效沟通,密切配合,充分授权
    解散阶段 Adjourning
    团队解散
激励理论

    优秀团队的特点:

    目标明确, 成员清除自己的工作对目标的贡献
组织结构清晰,岗位明确
成文货习惯的工作流程和方法内, 流程简明有效
明确的考核和评价标准, 工作结果公正公开,赏罚分明
共同制定并遵守组织纪律
协同工作,善于总结学习, 学习型组织

    输入

    人力资源管理计划
项目人员分派
资源日历

    工具技术

    人际关系技能
    培训
    人员技能不足,首先培训,其次调换岗位&nbsp;
    基本原则
    明文约定行为规范,沟通方式,协同工作,会议礼仪等, 以减少误解,提高生产力
集中办公
团队建设活动
认可和奖励
    人事评测工具
    洞察成员的优势和劣势,评估偏好和愿望

    输出

    团队绩效评价
事业环境因素更新

    管理项目团队
    Manage Project Team

    跟踪团队成员工作表现,提供反馈, 决绝问题并管理团队变更, 以优化项目绩效的过程.<br/>作用: 影响团队行为,管理冲突,解决问题,并评估团队成员的绩效

    输入

    人力资源管理计划
项目人员分派
团队绩效评价, 问题日志 工作绩效报告 组织过程资产

    工具技术

    观察和交谈
人际关系技能
    冲突管理
    产生原因
    不一致的需求
沟通不畅
稀缺资源竞争
进度优先级排序
个人工作风格
    解决方案
    合作 (解决问题) : 达成一致, 最好的方式,双赢,多数人接受
妥协(调节): &nbsp;双方让一步,一定程度满意,双输
强迫(命令) : 推行一方,否决一方, 嬴-输
缓和(包容) : 在和做之前,可以先适用缓和, 也要考虑冲突双方要不固执. 强调一致而非差异, 求同存异,缓和气氛
撤退(回避) : 当前不解决,未来解决的. 常用的做法有下次在开会讨论, 等信息更明朗时候决策等
    项目绩效评价
    技能的改进,从而使成员更搞笑的完成所分派的任务
能力和情感方面的改进, 从而提升团队能力,帮助团队更好的共同工作
团队成员流动率降低
增加团队凝聚力

    输出

    变更请求
项目管理计划更新
项目文件更新
事业环境因素更新
组织过程资产更新
